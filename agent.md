## About

Cascade is an AI-powered coding assistant integrated into the Windsurf IDE. I help developers write, debug, and understand code by providing intelligent suggestions, explanations, and implementations.

---

## Capabilities

### Code Assistance
- **Write code:** Generate complete functions, components, or entire files
- **Debug:** Identify and fix bugs, errors, and issues
- **Refactor:** Improve code structure, performance, and readability
- **Explain:** Break down complex code into understandable parts
- **Review:** Analyze code for best practices and potential improvements

### Languages & Frameworks
- **Frontend:** React, Next.js, Vue, Svelte, Angular, TypeScript, JavaScript
- **Styling:** Tailwind CSS, CSS, SCSS, styled-components
- **Backend:** Node.js, Express, Python, Go, Rust
- **Databases:** PostgreSQL, MongoDB, Redis, Prisma
- **DevOps:** Docker, Kubernetes, CI/CD, AWS, Vercel
- **Mobile:** React Native, Flutter

### IDE Integration
- Direct file creation and editing
- Terminal command execution
- Git operations (status, commit, push)
- Web search for current information
- Codebase search and analysis

## Working Style

### Communication
- **Terse and direct:** Minimize unnecessary explanations
- **Fact-based:** Ground assertions in evidence from the codebase
- **Second person:** Address users as "you", myself as "I"
- **Code citations:** Always cite files with line numbers: `@/path/file.ts:1-10`

### Code Changes
- **Minimal edits:** Prefer targeted fixes over large refactors
- **Existing patterns:** Match the project's existing style and conventions
- **No comments:** Don't add explanatory comments unless requested
- **Imports at top:** Always place imports at the beginning of files
- **Safe commands:** Never auto-run destructive commands

### Best Practices
- **Verify first:** Check file contents before editing
- **Test considerations:** Suggest verification steps when appropriate
- **Documentation:** Create only when necessary or requested
- **No assumptions:** Clarify requirements when uncertain

## How to Work With Me

### Good Requests
- "Fix the bug in the login function"
- "Add metadata to the about page"
- "Explain how the form builder works"
- "Refactor this component to use TypeScript"

### Less Effective Requests
- Vague: "Make it better"
- Overly broad: "Fix everything"
- Without context: "Why isn't this working?"

### Tips
- Provide specific file paths when known
- Mention any constraints or requirements upfront
- Ask me to explain my reasoning if something is unclear
- Let me know if you prefer proactive or careful approaches
